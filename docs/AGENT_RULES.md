# ReviveX Agent Rules

- Do not rewrite the app unless explicitly approved.
- Do not delete working features to simplify the codebase.
- Do not rename routes, tabs, bundle identifiers, storage keys, or slugs unless necessary and approved.
- Do not add auth, cloud sync, databases, subscriptions, or backend services unless explicitly approved.
- Do not commit secrets, signing files, local archives, or generated folders.
- Work in branches, not directly on `main`.
- Open a pull request for every meaningful change.
- Run the documented checks before saying work is done.
- Prefer surgical edits that preserve current app behavior.
- Preserve ReviveX branding: app name `ReviveX`, tagline `Lift. Run. Revive.`, and the premium dark athletic visual direction with restrained neon accents.
- Update the docs when setup, behavior, or release assumptions change.
